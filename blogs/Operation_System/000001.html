    # 从大学回忆开始

    在大学里面的操作系统课程内，老师提到编译源码再装载到内存里的过程大致为：

    # 软件编译过程

    使用命令：

    gcc HelloWorld.c -o HelloWorld 
gcc ./HelloWorld.c -o ./HelloWorld
    1
    2

    就可以编译这段代码。其实，GCC 只是完成编译工作的驱动程序，它会根据编译流程分别调用预处理程序、编译程序、汇编程序、链接程序来完成具体工作。

    前三种对应了大学老师提到的complie time，后者则是链接阶段，最后生成可执行文件 （ELF file）。

    Programs compiled on modern Linux systems place executable and shared code into ELF files (Executable and Linkable Format). ELF files have a strict structure.

    各部分的行为是：

    # 那么HelloWorld里面有什么？

    回想起大学里面老师教的程序分析，ELF文件由4部分组成，分别是ELF头（ELF header）、程序头表（Program header table）、节（Section）和节头表（Section header table）。

    # 看看ELF 头（ELF header）

    readelf -h HelloWorld
    1

    调重点的几个看看：

    1. Machine: Advanced Micro Devices X86-64

      文件编译运行的架构是AMD X86-64

    2. Magic -> ELF魔数，ELF字长，字节序，ELF文件版本等

    3. Type: DYN (Shared object file)

      文件种类是DYN（Shared object file - 共享目标文件）

    DYN类型意味着程序能装载在随机的地址，这让使程序像共享库一样在主存任何位置装载。这需要将程序编译成位置无关，并链接为ELF共享对象，提高了系统的安全性。这一点是由PIE(Position-Independent-Executable)实现的，在最近较新版本的GCC里面都会默认开启这个功能。我们可以通过执行gcc HelloWorld.c -o HelloWorld -no-pie 来关闭。这样我们跑出来的文件就是EXEC的。

    # 接着是程序头表（program header）

    readelf -l HelloWorld
    1

    显示程序头（段头）

    这是我当时上课做的笔记，对不同类型的程序头的描述：

    We have 13 program headers with five types, PHDR, INTERP, LOAD, DYNAMIC and NOTE type.

    • PHDR: Contains simple information about the program header, which is similarly to output of the ‘readelf -h HelloWorld’ flag.
    • INTERP: Specifies the location and size of a null-terminated path name to invoke as an interpreter, which cannot occur more than once in a file.
    • LOAD: Specifies a loadable segment, it will load the relevant information, such as header, code and data, to setup the program for executing.
    • DYNAMIC: Specifies dynamic linking information.
    • NOTE: Specifies the the location of notes (ElfN_Nhdr).

    可以看到我们程序的interpreter是/lib64/ld-linux-x86-64.so.2

    这是什么，又为什么需要这个interpreter呢？

    The interpreter in program is a loader, which will do all the setup work before the user runs the program. It will bring into all the required c program library files and map those into specific memory address. For example, the printf command is used from standard c library, which is required in somewhere of the memory to allow program to access. Hence, it needs to have an interpreter.

    接着我们可以知道我们的main函数被储存到哪个段（segment）内

    LOAD           0x0000000000001000 0x0000000000001000 0x0000000000001000
               0x00000000000001f5 0x00000000000001f5  R E    0x1000
    1
    2

    这是依靠E这个执行权限得到的，因为我们只有这一个main函数，所以必定会在该段内。

    # 接着就到了节（section)

    readelf -SW HelloWorld
    1

    # 节头表（Section header table）

    gcc -c HelloWorld.c
readelf -S HelloWorld.o
    1
    2

    # 跑一跑objdump

    objdump -d HelloWorld
    1

    我们先看<_start> section

    可以看到他开始的地址是0000000000001060，还记得最开始我们用readelf看的elf头吗？ELF头里面的entry point address也必然是这个。0x1060

    我们是怎么从<_start>到我们的main函数的？

    又是找回了上课的笔记

    The system will run the <_start> segment section first (the entry point). Next, it will do some packing up operations and use “callp” assemble command for passing control and jumping to C main library <__libc_start_main@GLIBC_2.2.5>, which occurs out the initialization of program. After initialing, c library will execute the program and pass the control pack to the main function. Thus, program will reach the main section.
