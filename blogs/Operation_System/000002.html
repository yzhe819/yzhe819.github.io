    # PC机的引导流程

    当 x86 PC 启动时，它执行的是一个叫 BIOS 的程序。BIOS 存放在非易失存储器中，BIOS 的作用是在启动时进行硬件的准备工作，接着把控制权交给操作系统。具体来说，BIOS 会把控制权交给从引导扇区（用于引导的磁盘的第一个512字节的数据区）加载的代码。引导扇区中包含引导加载器（CRUB引导程序）——负责内核加载到内存中。BIOS 会把引导扇区加载到内存 0x7c00 处，接着（通过设置寄存器 %ip）跳转至该地址。因为知识尚且不足，我们借用 GRUB 引导程序来引导我们的HelloOs。

    # 为什么是0x7c00? 内存里面发生了什么？

    简单说，计算机启动是这样一个过程。

    1. 通电
    2. 读取ROM里面的BIOS，用来检查硬件
    3. 硬件检查通过
    4. BIOS根据指定的顺序，检查引导设备的第一个扇区（即主引导记录），加载在内存地址 0x7C00
    5. 主引导记录把操作权交给操作系统

    所以，主引导记录就是引导"操作系统"进入内存的一段小程序，大小不超过1个扇区（512字节）

    0x7C00这个地址来自Intel的第一代个人电脑芯片8088 (opens new window)，以后的CPU为了保持兼容，一直使用这个地址。

    1981年8月，IBM公司最早的个人电脑IBM PC 5150上市，就用了这个芯片。

    当时，搭配的操作系统是86-DOS (opens new window)。这个操作系统需要的内存最少是32KB。我们知道，内存地址从0x0000开始编号，32KB的内存就是0x0000～0x7FFF。

    8088芯片本身需要占用0x0000～0x03FF，用来保存各种中断处理程序 (opens new window)的储存位置。（主引导记录本身就是中断信号INT 19h的处理程序。）所以，内存只剩下0x0400～0x7FFF可以使用。

    **为了把尽量多的连续内存留给操作系统，主引导记录就被放到了内存地址的尾部。**由于一个扇区是512字节，主引导记录本身也会产生数据，需要另外留出512字节保存。一旦操作系统被引导并开始，引导扇区将一直不会被使用指导重启，因此操作系统和应用程序可以自由的使用32KB的最后1kb（512 + 512B）空间。所以，它的预留位置就变成了：

      0x7FFF - 512 - 512 + 1 = 0x7C00
    1

    0x7C00就是这样来的。

    计算机启动后，32KB内存的使用情况如下。

    # 不同文件的功能：

    • entry.asm —— CRUB引导汇编代码

    • vgastr.c / vgastr.h —— 通过字符模式编程操纵显卡进而操控屏幕显示

    • main.c —— HelloOS的主程序（调用vgastr把Hello World打印到屏幕上）

    • hello.lds —— 链接脚本(linker script)

    • install.md —— 简要的安装指引

    • Makefile —— 构建软件的规则，Make工具能根据这些规则自动化构建软件

      makefile 文件中规则是这样的：首先有一个或者多个构建目标称为“target”；目标后面紧跟着用于构建该目标所需要的文件，目标下面是构建该目标所需要的命令及参数。与此同时，它也检查文件的依赖关系，如果需要的话，它会调用一些外部软件来完成任务。

    # 03. 安装

    GRUB 首先会显示启动项到屏幕，然后让我们选择启动项

    首先我们需要让GRUB知道Hello OS, 篡改grud

    GRUB 根据启动项对应的信息，加载 OS 文件到内存

    所以我们还需要知道boot目录挂载的分区，来让GRUB知道os文件的地址

    运行：

    df /boot/
    1

    其中的“sda1”就是硬盘的第一个分区，但是 GRUB 的 menuentry 中不能写 sda1，而是要写“hd0,msdos1”，这是 GRUB 的命名方式，hd0 表示第一块硬盘，结合起来就是第一块硬盘的第一个分区, 因此我们可以修改课程提供的代码为：

    menuentry 'HelloOS' {
     insmod part_msdos #GRUB加载分区模块识别分区
     insmod ext2 #GRUB加载ext文件系统模块识别ext文件系统
     set root='hd0,msdos1' #注意boot目录挂载的分区，第一块硬盘的第一个分区“sda1”
     multiboot2 /boot/HelloOS.bin #GRUB以multiboot2协议加载HelloOS.bin
     boot #GRUB启动HelloOS.bin
}
