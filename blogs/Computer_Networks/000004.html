    方式一：每两个主机之间构建一条线路 -> 需要大量的通讯线路

    方式二：使用交换设备 -> 当主机的数量较多的时候，所需吞吐量和接口花费巨大，难以直接实现。以及主机距离过大会导致难以连接到交换设备上 。（链路长度限制）

    # 3.0 IP数据报

    TCP/IP协议栈

    IP数据报格式

    固定部分的大小为20B

    首部长度：单位为4B，最小为5

    区分服务：通常用于传输优先级

    总长度：IP数据报的总长度，单位为1B

    生存时间（TTL）：用于避免无法传达的数据报在通讯网络中无限传输 - 占用资源

    协议：数据部分的协议

    首部检验和：只负责检验首部，用于检测发生变化后的TTL，标志或边偏移是否正确

    源地址和目的IP地址：长度都是32位

    可选字段：用于排错等

    填充：用于补全IP数据报的首部

    # 3.6.1 IPv6数据报格式

    相当于把IP数据报中的可变部分挪到了有效载荷中的选项部分（扩展首部）

    流标签：与IP数据报中的标识相似，但是相对于同一个流的一系列数据报都拥有同一个流标签，而IP数据报中的标识则是对一个数据报的分片来说，所有的分片都拥有同一个标识。

    有效载荷长度：只表示有效载荷的长度（包含扩展首部、数据）

    下一个首部：表示下一个扩展首部或者上层协议首部

    跳数：相当于IPv4的TTL

    # 4.1 RIP协议

    1. 仅和相邻路由器交换信息
    2. 路由器交换的信息是自己的路由表
    3. 每30秒交换一次路由信息，然后路由器根据新信息更新路由表。若超过180秒没有收到邻居路由器的信息，则判定邻居已经掉线，则更新自己的路由表
    4. 路由器最初工作的时候，只知道直接相连的网络的距离（为一），接着每一个路由器与相邻的路由器交换并更新路由信息，经过若干次更新后，所有路由器都会知道到达本自治系统任意一个路由器的最短距离和下级路由器的地址（收敛）
